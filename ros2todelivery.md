##ROS 2는 로봇 시스템에서 데이터를 주고받는 스마트 택배 회사로 비유할 수 있습니다. 
창고(노드)들이 규격화된 상자(메시지)를 통해 데이터를 주고받으며, 
배송 네트워크(DDS)와 배송 옵션(QoS)을 활용해 효율적이고 안전하게 운영됩니다. 
아래는 ROS 2의 주요 용어들을 택배 회사 비유로 나열한 표입니다.


| ROS 2 용어                     | 택배 회사 비유                     | 설명                                                                 |
|--------------------------------|------------------------------------|----------------------------------------------------------------------|
| Node                          | 창고                              | 독립적인 프로그램 단위, 데이터 주고받으며 역할 수행 (예: LiDAR 데이터 처리) |
| Topic                         | 정기 배송 서비스                  | 상자를 주기적 발송, 퍼블리셔가 보내고 구독자가 처리 (예: 센서 데이터) |
| Service                       | 즉시 배송 서비스                  | 요청-응답 방식 배송, 단일 작업에 적합 (예: 설정 변경)                 |
| Action                        | 추적 가능한 장거리 배송           | 목표 설정, 피드백, 취소 기능 제공 (예: 내비게이션)                    |
| Message                       | 규격화된 상자                     | 표준화된 데이터 포맷 (예: `sensor_msgs/Image`)                        |
| Interface                     | 상자 규격과 주문서 포맷          | 메시지(`.msg`), 서비스(`.srv`), 액션(`.action`) 정의                  |
| Publisher                     | 상자 발송자                      | 토픽으로 상자 발송 (예: 카메라 영상 상자)                           |
| Subscriber                    | 상자 수신자                      | 토픽 상자 받아 처리 (예: 내비게이션 LiDAR 데이터)                    |
| Client                        | 배송 요청자                      | 서비스 요청 상자 발송 (예: 로봇 초기화 요청)                         |
| Server                        | 배송 응답자                      | 서비스 요청에 응답 상자 발송 (예: 초기화 완료)                       |
| Action Client                 | 장거리 배송 요청자               | 액션 목표 상자 보내고 피드백/결과 수신 (예: 내비게이션 목표)         |
| Action Server                 | 장거리 배송 처리자               | 액션 목표 처리, 피드백/결과 상자 발송 (예: 로봇 이동)               |
| Parameter                     | 창고 운영 매뉴얼                 | 설정값, 동적 변경 가능 (예: 카메라 촬영 주기 10Hz)                  |
| DDS                           | 스마트 배송 네트워크             | 노드 간 상자 배송 자동 관리, 도메인 ID로 연결                        |
| Domain ID                     | 배송 네트워크 ID                 | 같은 네트워크 창고만 상자 주고받음 (예: ID 0번)                      |
| QoS                           | 배송 옵션                        | 속도/신뢰성 조정 (예: Best Effort=빠른 배송, Reliable=보험 배송)      |
| Package                       | 창고 설계도와 서류               | 노드, 메시지 정의, 설정 파일 포함 (예: 내비게이션 패키지)            |
| Message Definition            | 상자 규격 서류                   | 상자 데이터 구조 정의 (예: `sensor_msgs/PointCloud2`)                 |
| Service Definition            | 즉시 배송 주문서 규격            | 요청-응답 구조 정의 (예: `std_srvs/SetBool`)                         |
| Action Definition             | 장거리 배송 주문서 규격          | 목표, 피드백, 결과 구조 정의 (예: `nav2_msgs/NavigateToPose`)        |
| Launch File                   | 배송 스케줄표                    | 여러 창고 작업 동시 시작, Python/XML (예: 카메라+내비게이션 실행)    |
| ROS2 CLI                      | 배송 관리 콘솔                   | 배송 상태/창고 목록 확인 (예: `ros2 topic list`)                     |
| RQt                           | 배송 대시보드                    | 배송 흐름 시각화 GUI (예: `rqt_graph`)                               |
| RViz                          | 3D 배송 모니터                   | 로봇/상자 위치 3D 시각화 (예: LiDAR 데이터)                          |
| TF                            | 공통 주소 체계                   | 좌표 변환으로 위치 통일 (예: `base_link` → `map`)                    |
| URDF                          | 로봇 설계도                      | 로봇 물리적 구조 정의, XML (예: 바퀴/센서 위치)                      |
| Colcon                        | 물류 센터 건설 공장              | 패키지 빌드, 의존성 관리 (예: `colcon build`)                        |
| Ament                         | 창고 건설 규칙                   | 빌드 시스템 (`ament_cmake`, `ament_python`)                           |
| Bag                           | 배송 기록 장부                   | 상자 기록/재생 (예: `ros2 bag record`)                               |
| Logging                       | 배송 로그 시스템                 | 배송 과정 기록 (예: `rclpy.logging`)                                 |
| Intra-process Communication   | 창고 내부 배송                   | 같은 프로세스 내 빠른 상자 교환, 메모리 최적화                       |
| Component                     | 통합 창고                        | 여러 노드 프로세스 통합, 자원 효율화 (예: 센서+제어)                 |
| Lifecycle Node                | 창고 운영 상태 관리              | 노드 생애 주기 관리 (Configure, Activate, Shutdown)                  |
| SROS2                         | 배송 보안 시스템                 | 상자/주문서 암호화 (예: 인증서 보호)                                |
| Real-time                     | 긴급 배송 시스템                 | 정확한 시간 배송 보장, 실시간 OS 결합                                |
| REP                           | 택배 회사 표준 규정              | ROS 2 표준 정의 (예: REP-103 단위, REP-105 좌표)                     |
| Buildfarm                     | 대규모 물류 센터                 | 패키지 빌드/바이너리 배포 (예: ROS 2 패키지)                         |
| Unit Test                     | 배송 시스템 테스트               | 노드/상자 처리 검증 (예: `pytest`, `gtest`)                          |
| CI                            | 자동 품질 점검 로봇              | 코드 품질/빌드 자동 테스트 (예: GitHub Actions)                       |
| Lint                          | 배송 서류 점검기                 | 코드 스타일/오류 점검 (예: `ament_lint`)                             |
